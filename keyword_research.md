## KW選定基準

| 優先度 | 条件 |
|--------|------|
| 最優先 | CV度★★★ かつ 競合が弱い（DA30以下のサイトが上位） |
| 優先 | CV度★★ かつ 検索ボリューム月500以上 |
| 後回し | CV度★ または 大手メディアが独占（上位3件がリクルート・マネーフォワード等） |

### 狙いやすいKWの特徴
- 3語以上のロングテール（例：「副業 会社員 バレない 対策」）
- 比較・vs系（競合記事が古い or 比較表なし）
- 個別ツール名＋「使い方」「料金」「評判」

### 避けるべきKW
- 大手メディア（リクルート・マネーフォワード・弥生）が上位3件を独占
- 検索ボリューム月100未満の超ニッチ（初期フェーズは除く）
- YMYL領域（医療・法律・金融投資）

## 競合サイト

| サイト | 強み | 弱み（差別化余地） |
|--------|------|------------------|
| 比較.com / ITreview | 網羅性・DA高い | 実体験なし・比較表が浅い |
| マネーフォワードブログ | ブランド力 | 自社サービス誘導が前提 |
| 個人アフィリエイトブログ | 柔軟性 | 更新頻度低・E-E-A-T弱い |

### 差別化ポイント
- 実体験・職歴ベースのE-E-A-Tコンテンツ
- 比較表に推奨列・スコア・導入コストを数値で明示
- PR表記を隠さず読者との信頼関係を優先

## GSC・GA4 モニタリング指標

| 指標 | 確認頻度 | アクション基準 |
|------|---------|--------------|
| 検索順位（主要20KW） | 週次 | 圏外→30位以内を最初の目標 |
| クリック数・CTR | 週次 | CTR 2%未満はタイトル・メタ改善 |
| PV・滞在時間 | 月次 | 滞在2分未満は構成見直し |
| CVR（記事別） | 月次 | 1%未満はCTA改善対象 |

## KW調査・分析の指示テンプレート

---

### 1. 新規KW候補の調査
**使うタイミング**: 新しいテーマ・カテゴリに参入するとき

```
@keyword_research.md @seo_rules.md を読んで。
「[調査したいテーマ]」で新規KW候補を10本洗い出して。
各KWについて以下を整理して：
- 推定検索ボリューム（ラッコKW・Ubersuggest基準）
- 記事タイプ（比較/ランキング/個別LP/情報記事）
- CV度（★1〜3）
- 上位競合の強さ（弱/中/強）
- 既存KWリストとの重複確認
優先度順にランキング形式で出力して。
```

---

### 2. ロングテールKW展開
**使うタイミング**: メインKWが決まっていて、関連する記事を横展開したいとき

```
@keyword_research.md @seo_rules.md を読んで。
「[メインKW]」からロングテールKWを20本展開して。
条件：
- 3語以上の複合KW
- 検索ボリューム月100以上が目安
- 既存KWリストと重複しないもの
出力形式：KW / 記事タイプ / CV度 / 一言メモ
```

---

### 3. 競合記事調査
**使うタイミング**: 記事を書く前に上位を分析して差別化ポイントを探すとき

```
@keyword_research.md @seo_rules.md を読んで。
「[KW]」で上位5記事を調査して、以下を整理して：
- 各記事のタイトル・文字数・公開日
- H2・H3見出し構成
- 比較表の有無・質（列数・判定記号の有無）
- CTA種類と配置場所
- 不足している情報・古い情報（stacklogが勝てるポイント）
最後にstacklogが差別化すべき切り口を3点まとめて。
```

---

### 4. 検索意図の深掘り
**使うタイミング**: 記事構成を作る前に読者ニーズを整理したいとき

```
@keyword_research.md @seo_rules.md を読んで。
「[KW]」で検索する読者の意図を深掘りして。
以下の観点で整理して：
- 顕在ニーズ（検索した直接の目的）
- 潜在ニーズ（本当に解決したいこと）
- 不安・懸念（読者が気にしていること）
- 求める結論の形式（比較表/手順/ランキング/事例）
これをもとに最適な記事タイプと構成案（H2レベル）を提案して。
```

---

### 5. GSCデータ分析
**使うタイミング**: 週次・月次でGSCデータを貼り付けて改善優先度を決めるとき

```
@keyword_research.md @seo_rules.md を読んで。
以下のGSCデータを分析して、改善優先度順に整理して：
[GSCからコピーしたデータを貼り付け]

確認観点：
- 順位11〜30位でCTR低い記事 → タイトル・メタ改善候補
- 順位1〜10位でIMP多いのにCTR低い記事 → タイトル改善候補
- 順位圏外だが関連KWでIMP出ている記事 → コンテンツ追加候補
- クリック数が多いのに順位が安定しない記事 → 内部リンク強化候補
各カテゴリごとに上位3記事を抽出して、具体的な改善アクションをセットで出力して。
```

---

### 6. リライト優先度判断
**使うタイミング**: 既存記事の中でリライトすべき記事を絞り込むとき

```
@keyword_research.md @seo_rules.md を読んで。
以下の記事データからリライト優先度をA/B/C の3段階で判定して：
[記事タイトル・現在順位・月間PV・CVR・公開日をリスト]

判定基準：
A（最優先）: 順位11〜30位 + PV月100以上
B（優先）  : CVR 0% + PV月200以上 / 公開1年以上 + 順位30位以内
C（後回し）: 順位圏外 + PV月50未満 → 削除も検討
各記事に判定とその理由・具体的な改善アクションを付けて出力して。
```

---

### 7. 月次コンテンツ計画
**使うタイミング**: 翌月の記事スケジュールを組むとき

```
@keyword_research.md @seo_rules.md @affiliate_programs.md を読んで。
来月の記事投稿計画を作って。条件：
- 投稿本数：[N]本
- 優先カテゴリ：[ai-saas / fukugyou / 両方]
- KWリストの未着手分から選ぶ（状態が空欄のもの）
- CV度★★★を最優先、★★を次点
出力形式：
週次スケジュール（第1週〜第4週）
各記事：KW / 記事タイプ / 想定CV / 対応ASP案件
```

---

### 8. サイト全体の収益ポテンシャル試算
**使うタイミング**: 現状の記事構成から収益見込みを確認したいとき

```
@keyword_research.md @affiliate_programs.md を読んで。
現在のKWリスト20本が全て上位表示された場合の
月間収益ポテンシャルを試算して。

試算条件：
- 各KWの推定月間検索ボリュームをもとにPVを概算
- CTR: 順位1位=30% / 3位=15% / 5位=8% / 10位=3%
- サイト内CVR: 2%
- 各カテゴリの平均報酬単価はaffiliate_programs.mdを参照
カテゴリ別・記事タイプ別に分けて試算して。
最後に「月収10万円達成に必要な条件」をまとめて。
```
